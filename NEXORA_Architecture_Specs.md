# وثيقة المواصفات الفنية والهندسة المعمارية لنظام NEXORA
## منصة إدارة مكتبات الوسائط الذكية المخصصة للاستراحات وصالات الألعاب

---

## 1. الفكرة العامة للنظام (Core Concept)
نظام **NEXORA** هو نظام متكامل لإدارة وفهرسة وعرض مكتبات الوسائط الضخمة (أفلام، مسلسلات، أنمي، كرتون، وثائقيات) التي تصل أحجامها إلى عشرات أو مئات التيرابايت (100+ TB). 

بدلاً من التنقل التقليدي والمرهق داخل مجلدات وسائط معقدة على أقراص صلبة متعددة، يقوم النظام بفهرسة كافة الملفات برمجياً وبناء قاعدة بيانات ذكية تدعم البحث اللحظي، وتنسيق المحتوى تلقائياً بالاعتماد على أسماء الملفات وبصمات الفيديو (Video Fingerprinting)، وعرضها عبر واجهة سينمائية فاخرة وتفاعلية شبيهة بأشهر منصات البث العالمية (Netflix / Plex) ولكن ببيئة محلية بالكامل.

---

## 2. بنية النظام المعمارية: خادم مركزي وعملاء (Client-Server Architecture)
بما أن النظام مصمم للاستراحات وصالات الألعاب، فإن بنية النظام تعتمد على نموذج **الخادم المركزي والعملاء (Central Server & Clients)** لتحقيق أقصى درجات الاستقرار والكفاءة وتوفير مساحات التخزين.

### أ. الخادم المركزي (Central Server)
يتم تثبيت الخادم على كمبيوتر مركزي قوي متصل بشبكة الاستراحة المحلية (LAN) ويحتوي على كافة وسائط التخزين (الأقراص الصلبة HDDs). وظائفه تشمل:
1. **تخزين البيانات:** يحتوي على جميع ملفات الفيديو الأصلية والصور والأغلفة.
2. **محرك الفهرسة والمسح (Smart Scanner):** يراقب الأقراص محلياً، ويكتشف الملفات الجديدة، ويحلل أسماء الملفات ويستخرج البيانات الوصفية (Metadata).
3. **قاعدة البيانات والبحث:** يستضيف قاعدة البيانات المركزية ومحرك البحث الفوري.
4. **خادم البث (Media Streaming Server):** يقوم ببث الفيديو مباشرة عبر بروتوكول HTTP للأجهزة الفرعية دون الحاجة لتحميل الملف كاملاً، مع دعم الترميز المباشر (Transcoding) عند الحاجة عبر كارت الشاشة (GPU).
5. **الواجهة البرمجية (API Backend):** يوفر نقاط اتصال (endpoints) للعملاء لاستعلام المحتوى والبحث وتحديث الإحصائيات.

### ب. الأجهزة الفرعية / العملاء (Client Terminals)
هي أجهزة الكمبيوتر الخاصة بالزبائن في الاستراحة، أو شاشات التلفزيون الذكية المتصلة بالشبكة المحلية.
1. لا تحتاج هذه الأجهزة إلى مساحات تخزين إضافية، ولا تتطلب تحميل ملفات الفيديو بشكل مسبق.
2. تتصل بالخادم المركزي عبر شبكة الـ Wi-Fi أو الكابل (LAN).
3. تعرض واجهة المستخدم الفاخرة وتتيح البحث الفوري وتشغيل الفيديو عبر مشغل ويب حديث يدعم الترجمة، وتغيير مسارات الصوت، والتحكم بالجودة.

## 3. إعادة تقييم التقنيات واختيار البدائل لـ Flutter
بناءً على طلب العميل برفض استخدام Flutter لعدم ملاءمته للمشروع، تم استبدالها بتقنيات أكثر قوة، ومرونة، وسهولة في الصيانة والتشغيل على الشبكات المحلية:

### أ. لماذا تم استبعاد Flutter Desktop؟
1. **صعوبة التحديث والتثبيت:** استخدام Flutter Desktop يتطلب تثبيت كود برمجي (Executable) على كل جهاز فرعي بالاستراحة. أي تحديث للواجهة سيتطلب إعادة التثبيت على كافة الأجهزة يدوياً أو بناء نظام تحديث تلقائي معقد.
2. **الاستهلاك العالي للموارد:** تطبيقات Flutter Desktop قد تستهلك موارد المعالج والرام بشكل أكبر على الأجهزة الفرعية المتواضعة مقارنة بالويب الخفيف.
3. **دعم تشغيل الفيديو:** مشغلات الفيديو في Flutter Desktop ليست بمرونة وقوة مشغلات الويب المتطورة التي تدعم ميزات مثل الترجمات الخارجية المدمجة (Subtitles) وتغيير مسار الصوت (Audio Tracks) ديناميكياً بكل سهولة.

### ب. التقنيات البديلة المقترحة (The Modern Stack)

| الجزء | التقنية البديلة المقترحة | سبب الاختيار والميزات |
| :--- | :--- | :--- |
| **واجهة العميل (Frontend)** | **React.js + Tailwind CSS + Framer Motion** (عبر Vite) | 1. **بدون تثبيت (Zero Install):** يفتح العميل المتصفح فقط على عنوان السيرفر المحلي للاستمتاع بالنظام.<br>2. **تحديثات لحظية:** أي تعديل في السيرفر ينعكس فوراً عند جميع العملاء بمجرد عمل تحديث للصفحة.<br>3. **تصميم زجاجي فاخر:** يسهل جداً بناء الواجهة الزجاجية المطلوبة (Neon & Glassmorphism) بتأثيرات بصرية بالغة النعومة باستخدام Framer Motion.<br>4. **مشغل فيديو ويب متكامل:** استخدام مكتبات تشغيل مثل Video.js أو Plyr يمنح تحكماً كاملاً بالترجمات ومسارات الصوت المتعددة. |

## 4. مكونات ووظائف النظام بالتفصيل (Detailed Features & Modules)

### 1. وحدة المسح الذكي والفهرسة (Smart Scanner Module)
- **المراقبة الفورية (Live Watching):** مراقبة المجلدات المعينة في السيرفر باستخدام (File System Events). عند نسخ أي حلقة أو فيلم جديد للمجلد، يتم التقاطه فوراً وفهرسته دون الحاجة لإعادة المسح الشامل.
- **التنقيب عن التفاصيل الفنية:** استخراج الحجم، الامتداد، الجودة، مسارات الصوت والترجمة المتاحة وتخزينها كبيانات مصفاة.

### 2. وحدة الذكاء والتعرف التلقائي على الأسماء والوسائط (AI Metadata Matcher)
- **محلل التسمية الذكي (Smart Regex Parsing):** مكتبة متطورة تفهم وتفكك أسماء الملفات مثل:
  - `Attack.on.Titan.S04E05.1080p.mkv` ➜ الاسم: *Attack on Titan*، الموسم: *4*، الحلقة: *5*، الجودة: *1080p*.
  - `ون بيس الحلقة 1086 4k.mp4` ➜ الاسم: *ون بيس*، الموسم: *1* (تلقائي للأنمي الطويل)، الحلقة: *1086*، الجودة: *4K*.
- **التكامل مع قواعد البيانات العالمية:** جلب معلومات الغلاف (Posters)، والبانر، وقصة العمل، وتقييمات IMDB/TMDB/MyAnimeList وحفظها محلياً في السيرفر لتعمل بدون إنترنت (Offline Caching).
- **كاشف الأخطاء والملفات التالفة:**
  - تنبيهات الحلقات المفقودة (مثال: وجود الحلقات 1، 2، 4 يعني تنبيه للمسؤول بنقص الحلقة 3).
  - كشف ملفات الفيديو المعطوبة التي لا يمكن تشغيلها برمجياً وتحديدها للمسؤول لإعادة نقلها.
  - كشف وتصفية الملفات المكررة (Duplicates) بناءً على الحجم والاسم لتوفير مساحة التخزين المهدرة.

### 3. وحدة النقل والترتيب الاحترافية (Media Migration Wizard)
- **الهجرة الذكية:** شاشة معالجة مخصصة للمسؤول عند تركيب النظام لأول مرة. يقوم باختيار المجلدات العشوائية، ويقوم النظام بتحليلها بالكامل وعرض "تقرير المعاينة قبل التنظيم".
- **منظم الملفات المادي (Physical Organizer):** خيار لنقل الملفات وتعديل هيكليتها على القرص الفعلي تلقائياً لتصبح في مجلدات موحدة ومنظمة مثل:
  `Movies/Name (Year)/Name.mp4` أو `Anime/Name/Season XX/Episode XX.mkv`.
- **مدير النسخ المتطور (Local Copy Engine):** يتجاوز مدير نسخ ويندوز التقليدي، حيث يعرض سرعة النقل، نسبة الإنجاز، مؤقت متبقي، ودعم كامل لإيقاف واستئناف العمليات مع فحص سلامة الملف بعد النقل (Checksum validation).

### 4. لوحة الإحصائيات وإدارة الأقراص (Dashboard & Disk Manager)
- واجهة مخصصة لمدير الشبكة تعرض إجمالي استهلاك التخزين، صحة الأقراص (S.M.A.R.T)، حجم الأقسام (أفلام، مسلسلات، أنمي)، مع رسوم بيانية توضح مساحة الأقراص الممتلئة.

## 5. تصميم قاعدة البيانات المقترح (Proposed Database Schema)

يعتمد التصميم على علاقات واضحة لضمان السرعة القصوى في جلب البيانات:

```sql
-- جدول الأقسام الرئيسية
CREATE TABLE categories (
    id SERIAL PRIMARY KEY,
    name_ar VARCHAR(100) NOT NULL, -- أفلام، مسلسلات، أنمي...
    name_en VARCHAR(100) NOT NULL,
    slug VARCHAR(100) UNIQUE NOT NULL
);

-- جدول الأعمال (أفلام، مسلسلات، أنمي)
CREATE TABLE media_items (
    id SERIAL PRIMARY KEY,
    category_id INT REFERENCES categories(id),
    title_ar VARCHAR(255),
    title_en VARCHAR(255) NOT NULL,
    type VARCHAR(50) NOT NULL, -- 'movie', 'series', 'anime'
    plot_ar TEXT,
    plot_en TEXT,
    release_year INT,
    rating NUMERIC(3, 1),
    poster_path VARCHAR(500), -- مخزنة محلياً بالسيرفر
    banner_path VARCHAR(500),
    genres VARCHAR(255)[], -- مصفوفة من التصنيفات (أكشن، دراما...)
    status VARCHAR(50) DEFAULT 'completed', -- مستمر، مكتمل
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- جدول المواسم (للمسلسلات والأنمي)
CREATE TABLE seasons (
    id SERIAL PRIMARY KEY,
    media_item_id INT REFERENCES media_items(id) ON DELETE CASCADE,
    season_number INT NOT NULL,
    title_ar VARCHAR(150),
    title_en VARCHAR(150),
    UNIQUE(media_item_id, season_number)
);

-- جدول ملفات الفيديو (الحلقات أو الأفلام الفردية)
CREATE TABLE video_files (
    id SERIAL PRIMARY KEY,
    media_item_id INT REFERENCES media_items(id) ON DELETE CASCADE,
    season_id INT REFERENCES seasons(id) ON DELETE CASCADE, -- NULL للأفلام
    episode_number INT, -- NULL للأفلام
    title_ar VARCHAR(255),
    title_en VARCHAR(255),
    file_path TEXT NOT NULL, -- المسار الفيزيائي للملف في السيرفر D:\Media\...
    file_size BIGINT NOT NULL, -- الحجم بالبايت
    duration INT, -- المدة بالثواني
    resolution VARCHAR(50), -- 1080p, 4K, 720p
    video_codec VARCHAR(50), -- h264, hevc/h265
    audio_tracks JSONB, -- لغات الصوت المتاحة
    subtitles JSONB, -- ملفات الترجمة المدمجة والخارجية
    checksum VARCHAR(64), -- بصمة الملف للتحقق من التكرار والتلف
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- جدول الأقراص التخزينية المراقبة
CREATE TABLE storage_disks (
    id SERIAL PRIMARY KEY,
    disk_letter CHAR(1) UNIQUE NOT NULL, -- D, E, F...
    disk_label VARCHAR(100),
    total_space BIGINT NOT NULL,
    free_space BIGINT NOT NULL,
    is_active BOOLEAN DEFAULT TRUE,
    last_scanned TIMESTAMP
);
```

## 6. خطة التنفيذ وتوزيع المهام البرمجية (Implementation Roadmap)

خطة مقسمة لخطوات واضحة لتسهيل العمل لأي مطور أو ذكاء اصطناعي يبدأ بالمشروع:

### المرحلة 1: بناء البنية التحتية للخادم وقاعدة البيانات
1. إعداد خادم قاعدة البيانات PostgreSQL وبناء الجداول المطلوبة.
2. بناء الهيكل البرمجي لمشروع الخلفية (Go API Project).
3. إعداد خادم البحث Meilisearch وربطه مع تطبيق الخلفية.
4. كتابة كود مسح الأقراص الأساسي وقراءة الملفات (File Tree Scanner).

### المرحلة 2: محرك الذكاء والبيانات الوصفية (Metadata & Scraper Engine)
1. برمجة خوارزمية تحليل الأسماء (Regex Parser) لتغطية كافة الصياغات (العربية والإنجليزية).
2. ربط النظام مع TMDB API و MAL API لجلب صور الأفلام، البانرات والقصص تلقائياً وحفظها بالسيرفر.
3. دمج مكتبة MediaInfo/FFmpeg لقراءة الدقة والأبعاد والأكواد الفنية لكل ملف فيديو.
4. إعداد ميزة الكشف عن الحلقات المفقودة والملفات المكررة.

### المرحلة 3: بناء لوحة تحكم المسؤول وأدوات الترتيب (Admin Dashboard)
1. بناء واجهة React للمسؤول لمراقبة الأقراص والتخزين.
2. برمجة معالج الهجرة الذكي (Migration Wizard) مع واجهة عرض سيناريو التغيير قبل حدوثه.
3. تطوير محرك النسخ والنقل الداخلي الموثوق الداعم للاستكمال والـ Checksum.

### المرحلة 4: واجهة المستخدم النهائية (Client Web Application)
1. تصميم الواجهة الرسومية الفاخرة المعتمدة على Glassmorphism باللون البنفسجي والأزرق والأسود الفحمي.
2. تفعيل التصفح السلس للوسائط (أفلام، مسلسلات، أنمي) والبحث الفوري بواسطة Meilisearch.
3. دمج مشغل الفيديو المتقدم (المبني على HTML5) وتفعيل البث المحلي لملفات الفيديو والترجمات ديناميكياً من السيرفر.
4. إعداد نمط العرض التلفزيوني (Kiosk / TV Mode) لتسهيل استخدامه بالريموت أو أجهزة الكونسول.
